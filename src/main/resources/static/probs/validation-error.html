Validation Error

This application reports validation errors using an extension of Problem Details for HTTP APIs that adds the "errors" property, an array of objects with the following members:

  • "attribute" (string) - Name of the request body's attribute that caused the problem, if not present the whole body should be considered the cause of the problem.
  • "detail" (string) - A human-readable description of the error.

Example:

{
  "type" : "/probs/validation-error.html",
  "title" : "Bad Request",
  "status" : 400,
  "detail" : "Invalid request content.",
  "instance" : "/freelancers",
  "errors" : [ {
    "attribute" : "lastName",
    "detail" : "must not be blank"
  }, {
    "attribute" : "firstName",
    "detail" : "must not be blank"
  } ]
}

Find additional details in the Open API specification.
